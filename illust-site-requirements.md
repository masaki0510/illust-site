# イラストレーター・ポートフォリオサイト 要件定義書

最終更新: 2026-07-02
ステータス: ドラフト(デザイン方向性は本人ヒアリング待ち)

---

## 1. 目的

イラストレーター本人の実績を見せるポートフォリオサイト。合わせて、仕事の依頼を受け付ける窓口としても機能させる。

- **メイン目的**: 実績アピール(ギャラリー中心)
- **サブ目的**: 仕事の依頼受付(Googleフォーム経由)

### 活動概要

- 2年前から本格的に稼働中(実績・経験が一定蓄積されている)
- 専門領域: **Vtuberイラスト + Live2D の一括対応**
  - キャラクターデザイン/イラスト制作から、Live2Dでの動く化まで一気通貫で依頼できるのが強み
  - この「一括対応」自体がセールスポイントなので、サイト上でも明確に打ち出す
- 周辺サービス: Vtuber活動を補助するその他制作も対応
  - **三面図制作**(Live2D/モデリング用の資料としても、他のイラストレーターへの依頼資料としても使われるもの)
  - **ロゴ制作**(チャンネルロゴ、配信タイトルロゴなど)

## 2. サイト構成(確定)

| # | セクション | 内容 |
|---|-----------|------|
| 1 | ヒーロー | 名前・キャッチコピー・CTA(作品を見る/依頼する) |
| 2 | ギャラリー | 全作品を一覧表示。カテゴリー分けなし。Live2D作品は動く様子が伝わる見せ方を検討(動画/GIF埋め込みなど) |
| 3 | About | プロフィール、使用ツール |
| 4 | 依頼窓口 | 依頼メニュー紹介、Googleフォーム埋め込み |
| 5 | フッター | SNSリンク(X / Instagram / pixiv 想定) |

## 3. 確定している要件

- ギャラリーは**カテゴリー分けしない**(全部一覧でOK)
- 料金目安は**載せない**(問い合わせベース)
- 依頼受付は**Googleフォーム埋め込み**
- 画像はすぐ用意できる(数点〜数十点)
- 技術構成: 単一HTML(自己完結)+ GitHub Pages、Spirograph Breakと同じ運用フロー

### 4-0. ビジュアルコンセプト(確定)

**画廊(ギャラリー)風**を採用。

- 小さな個人ギャラリーの一室をイメージした世界観
- 木目のフローリング風背景に、作品を額装して余白多めに配置
- ホバー時に柔らかいスポットライトが当たったような光の演出
- 黒猫を「案内役」のワンポイントモチーフとして使用(セクション区切りやアイコンなどに、さりげなく登場させる)
  - 素材候補: 本人が過去に制作した**黒猫のLINEスタンプ**を活用できないか検討。複数ポーズ・表情がすでに用意されている想定なので、ページ内での簡単なアニメーション(切り替え表示、ホバーで違う表情になる等)の元絵として使いやすい
  - LINEスタンプ画像は背景透過のPNG(通常370×320px程度)であることが多いため、そのままサイトの装飾画像として流用しやすい。必要であれば本人からスタンプ画像一式をもらう
- 全体トーンは落ち着いて上品。原色ポップではなく、モネ風の淡い光の色使いをベースにする
- 梨(淡い黄緑〜クリーム系の色味)をアクセントカラーの参考にする
- Vtuber案件を依頼する側(企業/個人配信者)に信頼感を持たせる方向性を重視

### 4-7. 本人の好み(ヒアリング済みメモ)

以下は判明済みの好み。デザインの参考情報として活用する。

- **黒猫を飼っている**(好き)→ ワンポイントのマスコット/アイコンモチーフに使えそう
- **梨が好き**→ 色味(淡い黄緑〜クリーム)や、モチーフとしての梨のシルエット
- **木目調の家具が好き**→ 背景や区切りに木目テクスチャ、ぬくもりのある配色
- **モネが好き**→ 原色のポップというより、光が滲むような柔らかい色使い・筆致
- **魔女のスーパーマーケットが好き**→ 魔女・ファンタジー要素のモチーフとしても使えそう

これを踏まえると、当初案(原色ポップ・コルクボード風)よりも、**モネ風の淡い光の表現 × 木の温かみ × 黒猫のワンポイント**という方向性の方が本人らしさに近い可能性がある。次のデザイン案ではこちらの路線をベースに検討する。

- [ ] このイメージ(淡い印象派カラー+木目+黒猫モチーフ)が実際の好みと合っているか本人に確認
- [ ] 梨・黒猫をモチーフとして視覚的に使うことに抵抗がないか確認

## 5. 技術要件

- 単一 `index.html` に HTML/CSS/JS を内包(外部依存なし、Webフォントの利用は可)
- レスポンシブ対応(スマホでの閲覧が主想定)
- 画像は `images/` フォルダに配置し、後から差し替え・追加しやすい構造にする
- GitHub Pages にデプロイ(ブラウザのみでアップロード可能なワークフロー)
- キーボード操作・フォーカス表示など基本的なアクセシビリティに配慮
- **Live2D動画の扱い**: 「単一HTML完結」の原則と、動画ファイルの重さは相性が悪いため、以下のどちらかで対応
  - YouTube等に動画をアップロードし、`<iframe>` で埋め込む(サイト自体は軽いまま。おすすめ)
  - 動画ファイル自体をリポジトリに置いて `<video>` タグで直接再生(ファイルサイズに注意、GitHub Pagesの容量的にも本数が増えると厳しくなる可能性)
